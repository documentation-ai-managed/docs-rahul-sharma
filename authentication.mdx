## Overview

Secure your Rahul Sharma documentation space by implementing robust authentication and permission controls. Authentication verifies user identity, while permissions define what actions users can perform, such as reading, editing, or administering content.

You support multiple authentication methods to fit different use cases. Start with API keys for simple integrations, or use OAuth for user-based access.

<Callout kind="info">
Review your security settings before enabling public access to documentation pages.
</Callout>

## Generate an API Key

Follow these steps to create a new API key for programmatic access.

<Steps>
  <Step title="Navigate to Settings" icon="settings">
    Log in to your dashboard at `https://dashboard.example.com`.
    
    Select **Account > API Keys**.
  </Step>
  <Step title="Create Key" icon="plus">
    Click **Generate New Key**.
    
    Provide a descriptive name, such as `Docs Updater`.
  </Step>
  <Step title="Copy and Secure" icon="copy">
    Copy the generated key immediately. It will not be shown again.
    
    Store it securely in your environment variables as `DOCS_API_KEY`.
  </Step>
</Steps>

Use the key in API requests:

<CodeGroup tabs="JavaScript,cURL">
  ```javascript
  const response = await fetch('https://api.example.com/v1/docs', {
    headers: {
      'Authorization': `Bearer ${process.env.DOCS_API_KEY}`
    }
  });
  ```
  ```bash
  curl -H "Authorization: Bearer YOUR_API_KEY" \
       https://api.example.com/v1/docs
  ```
</CodeGroup>

OAuth provides token-based access for users.

<Request show-lines="true">
```javascript
const response = await fetch('https://auth.example.com/oauth/token', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    grant_type: 'client_credentials',
    client_id: 'YOUR_CLIENT_ID',
    client_secret: 'YOUR_CLIENT_SECRET'
  })
});
```
```bash
curl -X POST https://auth.example.com/oauth/token \
  -H "Content-Type: application/json" \
  -d '{
    "grant_type": "client_credentials",
    "client_id": "YOUR_CLIENT_ID",
    "client_secret": "YOUR_CLIENT_SECRET"
  }'
```
</Request>

## User Roles and Permissions

Assign roles to control access levels.

| Role       | Read | Write | Admin |
|------------|------|-------|-------|
| Viewer     | ✅   | ❌    | ❌    |
| Editor     | ✅   | ✅    | ❌    |
| Admin      | ✅   | ✅    | ✅    |

To assign roles:

```javascript
await fetch('https://api.example.com/v1/users/123/roles', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${process.env.DOCS_API_KEY}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({ role: 'editor' })
});
```

<Expandable title="Advanced Role Scopes" default-open="false">
Custom scopes allow granular control, such as `publish:pages` or `delete:spaces`.
</Expandable>

## Security Best Practices

Protect your documentation space with these guidelines.

<Callout kind="alert" title="Security Warning">
Never expose API keys in client-side code or public repositories.
</Callout>

- Rotate keys regularly using the dashboard.
- Use least-privilege roles.
- Enable IP allowlisting in settings.
- Monitor audit logs for suspicious activity.
