## Overview

Welcome to the Rahul Sharma documentation space. Use this centralized hub to organize, manage, and share all your project documentation efficiently. Whether you document codebases, APIs, user guides, or team processes, this space provides tools to create structured, searchable, and versioned docs.

You get a fully customizable MDX-powered site with components for rich content like code examples, diagrams, and interactive guides. Start by editing these pages or adding new ones to match your needs.

## Quick Start

Get up and running in minutes. Follow these steps to customize your docs.

<Steps>
  <Step title="Fork and Clone" icon="download" title-type="p">
    Fork this repository and clone it locally:

    ```bash
    git clone https://github.com/yourusername/rahul-sharma-docs.git
    cd rahul-sharma-docs
    ```
  </Step>

  <Step title="Install Dependencies" icon="package" title-type="p">
    Install with your preferred package manager:

    <CodeGroup show-lines="true" tabs={[]}>
      ```bash
      npm install
      ```

      ```bash
      yarn install
      ```
    </CodeGroup>
  </Step>

  <Step title="Start Development Server" icon="play" title-type="p">
    Launch the local server:

    ```bash
    npm run dev
    # or
    yarn dev
    ```

    Open `http://localhost:3000` to preview your docs.
  </Step>

  <Step title="Customize Content" icon="edit-3" title-type="p">
    Edit `.mdx` files in the `docs` folder. Add your pages, update frontmatter, and deploy.
  </Step>
</Steps>

## Brand Guidelines

Follow these guidelines to maintain consistency across your docs.

<Expandable title="Primary Brand Color" default-open="true">
  Use `{ #1B85FF }` as your primary brand color for headings, links, and accents. Apply it in CSS customizations:

  ```css
  :root {
    --brand-primary: #1B85FF;
  }

  a {
    color: var(--brand-primary);
  }
  ```
</Expandable>

<Expandable title="Typography and Icons" default-open="false">
  Stick to system fonts for readability. Use Lucide icons like `book-open`, `code`, and `zap` for visual appeal. Avoid custom fonts unless specified.
</Expandable>
